# 变更提案: 仓库整理（Phase 1：归档无用草稿到 old/）

## 需求背景
当前仓库目录存在大量历史快照、实验 UI（如 `ui_v2*`）与临时文件（如 Windows `.lnk`），实际运行只依赖 `tools/codex_thinking_sidecar/codex_thinking_sidecar/ui/` 与对应 Python 服务端。  
这些冗余内容混在主路径中会显著降低可维护性与定位成本，因此需要先做“最小风险”的整理：**不改变现有运行逻辑**，仅把明显无用/草稿/快照移入 `old/` 归档。

## 变更内容
1. 在仓库根目录新增 `old/`，作为本仓库的归档区（不参与运行）。
2. 将确定不参与当前运行的目录/文件移动到 `old/`（保持可回看）。
3. 更新 `.gitignore`：确保归档后的缓存目录仍保持忽略，避免误入版本控制。
4. 更新知识库：记录整理范围与后续 Phase 2 的迁移方向（项目结构扁平化/改名）。

## 影响范围
- **模块:** 目录结构（归档）/ 文档（知识库）
- **文件:** `.gitignore`、归档目录与知识库记录
- **API:** 无
- **数据:** 无

## 核心场景

### 需求: 仓库目录“变干净”且不影响当前运行
**模块:** 仓库结构

#### 场景: 仍可通过现有入口启动并访问 UI
保持 `run.sh`/`ui.sh` 现有用法不变；运行侧仍以 `/ui` 作为主 UI，不引入新的依赖与构建步骤。

## 风险评估
- **风险:** 误移动“当前运行必需文件”导致启动失败或 UI 404
- **缓解:** 仅移动明确标记为快照/草稿/实验产物；执行后做最小冒烟验证（`/health`、`/ui`）。
